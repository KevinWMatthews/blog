What is this blog all about?

This series of posts is about good practices in software development.

It has examples of design patterns, testing strategies, language semantics, and third-party libraries. This is a collection of what I've had occasion to learn, so there is a bias toward embedded systems.

The goal of each post is to actually "do" the thing in question, so code examples are part of example projects.
